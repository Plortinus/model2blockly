Recommended Next Steps for an Ecore Developer

  1. Open the generated *_standalone.html file and create a small model.
  2. Check this report's EClass, EAttribute and EReference mappings.
  3. If the structure is correct but the blocks are hard to read, add only presentation annotations such as source="blockly" message0, colour, category or tooltip.
  4. If labels in reference dropdowns are not meaningful, add source="ui" referenceLabelField=....
  5. Rerun the same Eclipse launch configuration and compare the new report.
