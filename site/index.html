Standard generation pipeline

The Ecore input is transformed into a formal EMF intermediate model before the Blockly editor is emitted.

1. Source model

Reuse or create an annotated Ecore metamodel.

2. Intermediate XMI

The model is converted to the generated EMF EditorSpec, written as intermediate/*_blocklyspec.xmi, then reloaded and validated.

3. Blockly editor

The reloaded intermediate model generates Blockly blocks, toolbox, validation runtime, reports, and HTML.

4. Domain instance XMI

The generated editor exports user-created blocks as JSON or domain XMI in the source namespace.

Model-driven workflow

The project keeps Ecore as the source of truth and makes the generated EMF model visible before producing Blockly code.
