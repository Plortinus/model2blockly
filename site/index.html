  3. 3

    Generate an editor

    Right-click a .model2blockly or .ecore file and run Generate Blockly Editor.

Standard generation pipeline

Both input routes use the same formal intermediate model before the Blockly editor is emitted.

1. Source model

Author a .model2blockly file or reuse an annotated Ecore metamodel.

2. Intermediate XMI

The model is converted to the generated EMF EditorSpec, written as intermediate/*_blocklyspec.xmi, then reloaded and validated.

3. Blockly editor

The reloaded intermediate model generates Blockly blocks, toolbox, validation runtime, reports, and HTML.

4. Domain instance XMI

The generated editor exports user-created blocks as JSON or EMF-style XMI in the source domain namespace.

Choose your modeling route

Both routes generate the same kind of Blockly editor. Pick the input model that matches your project.

Use .model2blockly when you want a concise text model

Best for new models, hand-written examples, tutorials, and quick editor experiments.

  • Readable syntax for classes, fields, inputs, and validation.
  • Editor diagnostics report generator-facing issues before generation.
Open DSL guide Example model

Use Ecore when you already have an EMF metamodel

Best for existing EMF projects or models that need Ecore IDs, opposites, packages, and annotations.

  • Uses standard EAnnotation metadata for Blockly, UI, code, and validation.
  • Preserves EMF concepts such as containment, bounds, IDs, and eOpposite.
